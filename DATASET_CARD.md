# Описание набора данных

## Краткое описание
Набор данных содержит посты из трёх телеграм-каналов, связанных с московским метрополитеном. Данные предназначены для тематического моделирования и поиска скрытых тем.

## Состав набора данных

### Исходная структура
Файл `raw.csv` содержит следующие столбцы:

1. **`message_id`** — уникальный идентификатор сообщения в Telegram
2. **`text`** — основной текст поста (ключевой признак для анализа)
3. **`date`** — дата и время публикации
4. **`views`** — количество просмотров поста
5. **`reactions`** — реакции на пост 
6. **`comments`** — количество комментариев
7. **`channel_name`** — название телеграм-канала
8. **`word_count`** — количество слов в тексте

### Размеры данных
- **Всего записей**: 31705
- **После обработки**: 12434 
- **Количество слов в превом эксперименте**: 443676
- **Количество слов после дополнительной обработки**: 438491

### Источники данных
1. **"Подслушано Метро Москвы"** — неофициальный канал пассажиров
   - Тип контента: жалобы, вопросы, обсуждения
   - Стиль: неформальный, разговорный
   - Целевая аудитория: пассажиры метро

2. **"Дептранс. Оперативно"** — официальный канал Департамента транспорта Москвы
   - Тип контента: официальные объявления, отчеты, оповещения
   - Стиль: формальный, официальный
   - Целевая аудитория: все пользователи транспорта

3. **"Московский метрополитен"** — официальный канал метрополитена
   - Тип контента: новости, анонсы, исторические факты
   - Стиль: формальный, официальный
   - Целевая аудитория: пассажиры метро

## Предобработка

В проекте используются три версии данных:

1. **`raw.csv`** — исходные необработанные данные (⚠️ заранее удалены дубликаты и пропуски)
2. **`data_processed2.csv`** — обработанные данные, готовые для обучения
3. **`data_without_stopwords.csv`** — финальный датасет с дополнительным удалением специфичных стоп-слов

## Стратегия разделения данных

Для задач тематического моделирования данные не разделяются на train/test, так как:

1. Тематическое моделирование — задача unsupervised learning
2. Все данные используются для построения тематической модели
3. Качество оценивается по семантической когерентности и интерпретируемости
