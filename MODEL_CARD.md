# Описание моделей, использованных в проекте по тематическому моделированию в области московского метро

## Модель: BERTopic для анализа тематики постов о московском метро

## Назначение модели

Модель предназначена для автоматического выявления и анализа основных тем обсуждений в телеграм-каналах, связанных с московским метрополитеном.

### Основные задачи:
1. Кластеризация тем: группировка постов по семантически близким темам
2. Визуализация: представление взаимосвязей между темами и их распределения
3. Инференс: классификация новых постов по существующим темам

## Конфигурация модели (bertopic.yaml):

#### Эмбеддинг и векторное представление:
- **Эмбеддинг-модель**: `sentence-transformers/paraphrase-multilingual-MiniLM-L12-v2`
- **Векторизатор**:
  - Диапазон n-грамм: `[2, 3]` (биграммы и триграммы)
  - Максимальное количество признаков: `5000`
  - Язык: `russian`
#### UMAP (уменьшение размерности):
- Количество соседей: `100`
- Целевая размерность: `5`
- Минимальное расстояние: `0.0`
- Метрика: `cosine`
- Random state: `42`
#### HDBSCAN (кластеризация):
- Минимальный размер кластера: `50`
- Минимальное количество выборок: `20`
- Эпсилон выбора кластера: `0.1`
- Метрика расстояния: `euclidean`
- Метод выбора кластера: `eom` (Excess of Mass)
#### Параметры BERTopic:
- Минимальный размер темы: `50`
- Количество тем: `auto`
- Расчет вероятностей: `true`
- Режим подробного вывода: `true`

## Тип модели и подходы

### Основная модель: **BERTopic** (для продакшена)
- Подход: современный метод тематического моделирования на основе BERT
- Архитектура: могоэтапный пайплайн с семантическими эмбеддингами
- Статус: модель для инференса и API

### Экспериментальные модели: **LDA** (3 варианта)
- LDA_v1: Базовый LDA с параметрами по умолчанию
- LDA_v2: Оптимизированный LDA с подобранными гиперпараметрами
- LDA_v3: LDA с расширенным словарем и специальной предобработкой
- Статус: только для экспериментов и сравнения подходов

## Входные данные

### Формат данных:
- Файл: `data/data_without_stopwords.csv`
- Ключевая колонка: обработанный текст
- Язык: русский
- Длина: короткие тексты (в среднем 10-50 слов)

## Общая схема проекта:

Сбор данных → Предобработка → Эксперименты (LDA) → Продакшен (BERTopic) → API сервис

## Валидация:

Автоматическая: метрики когерентности и кластеризации
Ручная: Интерпретация тем экспертом
